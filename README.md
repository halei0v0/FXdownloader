# FXdownloader

多源小说下载工具，支持蚂蚁文学、思兔阅读、顶点小说、笔下文学、铅笔小说、海棠文学、番茄小说七大源，多源自动搜索、相关性排序、分工加速下载、三层失败重试、缺失内容自动补全。

## 功能特点

### 核心下载

- **多源聚合搜索** - 输入书名即可同时搜索七大源，无需手动切换，自动给出所有可用结果
- **相关性排序** - 搜索结果按标题相关性自动排序：完全匹配 > 前缀匹配 > 包含匹配，同分时主源优先、有封面优先
- **多源分工下载** - 主源下载失败时自动切换其他源重试，按章节标题跨源匹配
- **三层失败重试** - 单章失败时先在相同源重试（3 次退避）→ 跨源重试（按标题匹配）→ 缺失补缺，最大化下载成功率
- **失败章节自动选中** - 所有重试仍失败的章节会自动勾选并以红色高亮，滚动定位到首个失败章节，方便手动重试
- **章节去重排序** - 自动剔除章节列表开头倒序的"最新章节"，正确排序
- **字体解密** - 自动解密番茄小说的字体加密内容
- **批量下载** - 支持选择章节范围批量下载，并发加速

### 发现与浏览

- **排行榜** - 从速读谷 (sudugu.org) 获取真实热门排行榜（前 30 名，3 页），按天缓存避免反爬
- **分类浏览** - 7 大分类（玄幻/仙侠/都市/历史/军事/科幻/言情），数据同样来自速读谷并按天缓存
- **点击即搜** - 排行榜和分类中的小说点击后，自动用书名在各源搜索可用结果
- **封面缓存** - 小说封面获取一次后自动保存到本地，搜索结果、收藏列表均自动显示封面，后续不再重复请求
- **封面缓存管理** - 设置面板显示封面缓存条目数和占用大小，可一键清除
- **小说状态显示** - 搜索结果卡片和详情页均显示完本/连载状态标签（绿色完本/橙色连载），自动从源解析并统一映射

### 下载管理

- **下载进度预估** - 实时显示下载进度、速度和剩余时间预估
- **暂停/续传** - 支持随时暂停下载任务，之后从断点续传，不重复下载已完成章节
- **章节更新** - 一键检查并下载小说的新增章节，同时检测并补全上次未完成的内容
- **历史记录** - 自动记录下载历史，可快速重新打开
- **收藏夹** - 收藏喜爱的小说，卡片支持直接点击"阅读"按钮进入阅读器
- **阅读功能** - 内置阅读器，支持连续滚动阅读（章末自动加载下一章，同页面无缝衔接）

### 阅读体验

- **连续滚动阅读** - 阅读到章节末尾时自动加载下一章并追加到当前页面，无需跳转等待
- **护眼背景** - 默认开启护眼模式（暖色纸质感），可一键切换深色背景
- **字号自动保存** - 字号范围 12-36 px 自由调节，调整后自动保存到配置，下次打开自动恢复
- **阅读进度记忆** - 自动记录最后阅读位置，下次打开直接定位到上次章节
- **章节快速跳转** - 侧栏章节列表实时高亮当前章节，点击即跳转

### 界面

- **现代化 WebUI** - 基于 pywebview + HTML/CSS/JS 的深色主题界面
- **初始发现页** - 启动即显示排行榜和分类，点击小说后平滑切换到详情视图
- **购物篮批量下载** - 将多本小说加入购物篮，一键批量下载
- **应用图标** - 支持自定义应用图标（打包 EXE 与开发模式窗口均显示），兼容新旧版本 pywebview
- **EXE 打包** - 支持 PyInstaller 一键打包为独立可执行文件
- **开发数据清理** - 内置清理脚本，一键清除数据库/下载文件/缓存/配置等开发测试数据

## 环境要求

- Python 3.8+
- Windows / Linux / macOS
- Edge 浏览器驱动（仅番茄官网登录需要）

## 安装步骤

1. 克隆项目：
```bash
git clone https://github.com/halei0v0/FXdownloader.git
cd FXdownloader
```

2. 安装依赖：
```bash
pip install -r requirements.txt
```

3. 安装 Scrapling（可选，用于多源爬取，推荐 editable 安装）：
```bash
pip install -e Scrapling-main
```

## 使用方法

### WebUI 模式（推荐）

**Windows：** 双击 `启动WebUI.bat`

**命令行：**
```bash
python web_ui/app.py
```

打开后自动启动本地服务器并弹出窗口。界面操作：

1. **发现页** - 启动后显示排行榜和分类，点击小说进入详情
2. **搜索** - 顶部输入书名或小说 URL，点击搜索，结果聚合所有源
3. **下载** - 选择小说后获取章节列表，勾选要下载的章节（默认全选）
4. **更新小说** - 点击"检查更新"按钮，自动检测新增章节和缺失内容，仅选中需要下载的章节
5. **管理** - 通过左侧导航访问历史记录、收藏夹、暂停任务
6. **批量** - 将多本小说加入购物篮，一键批量下载

### GUI 模式（旧版 tkinter）

```bash
python gui.py
```

### 命令行模式

```bash
# 通过 URL 下载
python main.py download https://fanqienovel.com/page/711914860

# 通过小说 ID 下载
python main.py download 711914860

# 下载指定章节范围
python main.py download 711914860 --start 1 --end 50

# 搜索小说
python main.py search 诡秘之主

# 列出已下载的小说
python main.py list

# 导出已下载的小说
python main.py export 711914860
```

### 打包为 EXE

**Windows：** 双击 `打包.bat`

**命令行：**
```bash
pyinstaller FXdownloader.spec
```

打包后的可执行文件位于 `dist/FXdownloader.exe`。

**新增功能**

- 新增思兔阅读源 (sto66.com)：支持搜索，无需登录，章节列表自动分页抓取（每页 500 章，自动检测"下一页"），书籍信息解析作者/分类/字数/状态/封面

- 小说状态显示：搜索结果卡片右上角显示"完本"（绿色）/"连载"（橙色）状态标签，详情页标签同步显示
- 状态字段统一映射：前端 `formatNovelStatus()` 将各种状态文案（完结/已完结/全本 → 完本；连载中/更新中 → 连载）归一化显示，正确处理"未完结"/"未完本"否定前缀避免误判

**优化改进**

- 后端 `_novel_info_to_dict` 和 `search_all_sources` 将 `status` 从 `extra` 提取到顶层字段，便于前端统一访问
- sto66 源 URL 解析使用 `re.search` 而非 `re.match`，兼容绝对 URL
- sto66 源状态 class 检测支持多 class（`'red' in cls.split()` 而非精确匹配）
- 多源搜索并发调用七大源（蚂蚁文学、思兔阅读、顶点小说、笔下文学、铅笔小说、海棠文学、番茄小说）
- 前端源下拉框新增"思兔阅读"选项，与后端 `list_sources()` 顺序保持一致
- 排行榜和分类页不显示小说状态标签：因速读谷 (sudugu.org) 缓存的状态可能与实际小说源不一致，改为只在点击小说进入详情页后显示从实际源获取的准确状态（`buildNovelCard` 新增 `options.hideStatus` 参数）

**Bug 修复**

- 修复章节列表获取不全的问题：原 `_dedup_and_sort_chapters` 先去重后剔除倒序段，导致同时出现在倒序段和正序主体中的最新章节被永久丢失（蚂蚁文学《末日生存方案供应商》丢失最新 9 章）。改为先检测并跳过倒序段，再对正序主体去重
- 修复思兔阅读源"未知源"错误：前端源下拉框 `<select id="sourceSelect">` 缺少 `<option value="sto66">`，导致 `selectNovel()` 中 `sourceSelect.value = 'sto66'` 静默失败，`loadChapters()` 读取到错误的源 key 传给后端，返回"未知源"。补全下拉框选项后修复
- 修复状态标签误判：`formatNovelStatus` 用子串匹配且 `finishKeywords` 先检查，导致"未完结"/"未完本"包含"完结"/"完本"子串被误判为"完本"。改为优先检查否定前缀关键词
